# DummyClient #1

더미 클라이언트를 만들게 되면 라이브 전 실제 라이브에서 발생할 수 있는 버그들을 잡아낼 수 있는 테스트를 할 수 있다.

클라에서 DummyClient를 만들 수 있기는 하지만 그렇게 되면 코드들이 뒤섞일 수 있게 된다. 예를 들면 PacketHandler 내부에서 더미 클라이언트 테스트 일 때와 아닐 때를 분기 처리를 하는 식이다. 실제 중요한 코드와 테스트용 코드가 뒤섞이는 게 기분이 좋지 않다.

일단 1차적으로 많은 플레이어들이 Server에 붙어서 Broadcasting을 하는지 여부를 테스트 해보자

따라서 우선적으로 콘텐츠와 관련된 부분은 당장 작업을 하지 말고 필요할 때 추가를 해보자.
