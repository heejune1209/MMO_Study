# Data & Config

캐릭터의 스탯, 스킬 정보 등을 json 파일로 뽑아서 관리하는 방법으로 진행

왜 서버와 클라 각각 데이터와 관련된 작업을 따로 해야하는 것일까?

DataManager가 클라와 서버가 똑같지 않은 경우가 있게 된다.

아이템을 강화하는 확률을 예를 들면 해당 아이템 정보를 클라와 서버가 각각 들고 있게 될 것이고, 

클라를 까보게 되면 강화 확률이 어떻게 되는지 눈치를 챌 수도 있다.

따라서 경우에 보안을 위해 클라에서 사용하는 데이터 정보와 서버에서 사용하는 데이터의 정보를 따로 관리하는 경우가 많다.

현재는 클라에서 데이터 원본을 들고 있고 해당 정보를 서버에서 가지고 오는 형식으로 하고 있는데 

이것을 별도의 Common 폴더를 만들어서 거기에 데이터 원본을 두고 클라와 서버에서 관리하는 방식으로 작업을 진행해줘야 한다.

```protobuf
enum SkillType {
  SKILL_NONE = 0;
  SKILL_AUTO = 1;
  SKILL_PROJECTILE = 2;
}
```

```json
"id": "1",
"name": "평타",
"cooldown": "0.2",
"damage" : "10",
"skllType": "SkillAuto"
```

protobuf enum을 위와 같이 설정을 하면 실제로 SkillAuto 가장 앞자리가 대문자로 바뀐 형태로 저장이 된다.
